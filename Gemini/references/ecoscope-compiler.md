# wt-compiler Reference

Reference for the `wt-compiler` tool used to scaffold and compile ecoscope workflows.

---

## Installation

```bash
pixi global install -c https://prefix.dev/ecoscope-workflows -c conda-forge wt-compiler --run-post-link-scripts
```

Installed location: `/home/gitonga/.pixi/envs/wt-compiler/`

`wt-compiler` is available globally in PATH — no conda/pixi environment activation needed.

---

## Architecture

`wt-compiler` uses ephemeral rattler environments to discover tasks:

1. Reads `spec.yaml` requirements (packages + channels)
2. Solves and installs a temporary environment containing the specified task libraries
3. Calls `wt-registry` CLI to get task signatures from that environment
4. Generates workflow artifacts (DAG code, JSON schemas, pixi.toml, Dockerfile, etc.)

This means no separate compile environment in the wt-* repo's `pixi.toml` is needed — compilation is fully self-contained in wt-compiler.

## Key Commands

### Scaffold New Workflow

Interactive mode:
```bash
wt-compiler scaffold init
```

Non-interactive mode:
```bash
# New format (ecoscope-platform >= 2.11 + PGAFF custom library)
wt-compiler scaffold init --no-interactive \
  --workflow-id <id> \
  --workflow-name "<name>" \
  --author-name "<author>" \
  --output-dir <dir> \
  --requirements '{"name":"ecoscope-platform","version":">=2.11.0,<3","channel":"https://repo.prefix.dev/ecoscope-workflows/"}' \
  --requirements '{"name":"ecoscope-workflows-ext-custom","version":">=0.1.0,<1.0.0","channel":"https://repo.prefix.dev/ecoscope-workflows-custom/"}'

# Legacy format (ecoscope-workflows-core + ext-ecoscope)
wt-compiler scaffold init --no-interactive \
  --workflow-id <id> \
  --workflow-name "<name>" \
  --author-name "<author>" \
  --output-dir <dir> \
  --requirements '{"name":"ecoscope-workflows-core","version":">=0.22.18,<0.23.0","channel":"https://repo.prefix.dev/ecoscope-workflows/"}' \
  --requirements '{"name":"ecoscope-workflows-ext-ecoscope","version":">=0.22.18,<0.23.0","channel":"https://repo.prefix.dev/ecoscope-workflows/"}' \
  --requirements '{"name":"ecoscope-workflows-ext-custom","version":">=0.0.46,<0.1.0","channel":"https://repo.prefix.dev/ecoscope-workflows-custom/"}'
```

Note: `wt-compiler scaffold init` creates a directory named after `--workflow-id`. Since PGAFF repos use `wt-<name>` convention (hyphens), rename after scaffolding:
```bash
mv "$WORKFLOW_ID" "wt-$NAME"
```

### Compile Workflow

```bash
wt-compiler compile --spec spec.yaml \
  --pkg-name-prefix=ecoscope-workflows \
  --results-env-var=ECOSCOPE_WORKFLOWS_RESULTS \
  --clobber \
  [--update] \
  [--install]
```

---

## Environment Variables

| Var | Used by | Value |
|-----|---------|-------|
| `ECOSCOPE_WORKFLOWS_RESULTS` | PGAFF workflows (`--results-env-var`) | `file:///path/to/results/` |
| `WT_RESULTS` | SDK default / wt framework | `file:///path/to/results/` |

PGAFF uses `ECOSCOPE_WORKFLOWS_RESULTS` via `--results-env-var=ECOSCOPE_WORKFLOWS_RESULTS` at compile time.

---

## Compile Flags

| Flag | Description |
|------|-------------|
| `--clobber` | Overwrite existing output directory |
| `--update` | Carry over lockfile from existing build (faster iterative development) |
| `--install` | Generate lockfile and install after compilation |
| `--pkg-name-prefix` | Prefix for generated package — use `ecoscope-workflows` for PGAFF |
| `--results-env-var` | Env var for results URL — use `ECOSCOPE_WORKFLOWS_RESULTS` for PGAFF |

## Generated Output Structure

```
ecoscope-workflows-<id>-workflow/
├── ecoscope_workflows_<id>_workflow/
│   ├── __init__.py
│   ├── cli.py              # Click CLI: run, get, convert
│   ├── dispatch.py         # Routes to correct DAG executor
│   ├── formdata.py         # Pydantic model for form data
│   ├── metadata.py         # Schema utilities
│   ├── params.py           # Pydantic model for parameters
│   ├── params.json         # Flat JSON Schema
│   ├── rjsf.json           # React JSON Schema Form config
│   ├── response.py         # Response model
│   └── dags/
│       ├── jupytext.py           # Notebook format
│       ├── run_async.py          # Async execution
│       ├── run_async_mock_io.py
│       ├── run_sequential.py
│       └── run_sequential_mock_io.py
├── tests/
│   ├── conftest.py
│   ├── test_metadata.py
│   └── test_results.py
├── pixi.toml       # Autogenerated — do not edit manually
├── Dockerfile
├── graph.png       # DAG visualization
└── README.md
```

## DAG Execution Modes

| Mode | File | Description |
|------|------|-------------|
| `jupytext` | `jupytext.py` | Interactive notebook |
| `async` | `run_async.py` | Parallel with dependency graph |
| `sequential` | `run_sequential.py` | Linear execution |
| `*_mock_io` | `*_mock_io.py` | Testing with mocked I/O |

## Variable Parsing in spec.yaml

Variables are parsed into typed objects:

| Syntax | Parsed Type |
|--------|-------------|
| `${{ workflow.<id>.return }}` | `TaskIdVariable` |
| `${{ env.<VAR> }}` | `EnvVariable` |
